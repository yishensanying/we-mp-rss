# Web UI 环境异常统计使用指南

## 功能概述

环境异常统计页面提供了可视化的界面来监控和分析微信公众号文章获取过程中的环境异常情况。

## 访问页面

### 方式 1: 通过导航菜单

1. 登录 Web UI
2. 点击顶部导航栏的 "异常统计" 菜单项
3. 进入环境异常统计页面

### 方式 2: 直接访问 URL

```
http://your-domain/env-exception
```

**注意**: 需要管理员权限才能访问此页面

## 页面功能

### 1. 统计概览

页面顶部显示四个关键指标：

- **异常总数**: 当日环境异常的总次数
- **异常URL数**: 出现异常的文章URL数量
- **受影响公众号**: 出现异常的公众号数量
- **统计日期**: 当前查看的日期

### 2. 日期选择

可以使用日期选择器查看历史统计数据：

1. 点击日期选择器
2. 选择要查看的日期（不能选择未来日期）
3. 点击 "查询" 按钮
4. 或点击 "今日统计" 快速查看今天的数据

### 3. 公众号异常统计

以表格形式展示各公众号的异常情况：

- **公众号ID**: 出现异常的公众号标识
- **异常次数**: 该公众号出现异常的总次数
- **占比**: 占总异常次数的百分比

表格支持排序，方便快速找到异常最多的公众号。

### 4. 异常URL列表

列出所有出现异常的文章URL：

- **URL**: 文章链接（可点击打开）
- **异常时间**: 出现异常的时间戳

支持分页浏览，方便查看大量数据。

### 5. 最近异常日志

以列表形式展示最近20条异常记录：

- 公众号名称和ID
- 异常时间
- 文章URL（可点击）

## 操作说明

### 刷新数据

点击 "刷新" 按钮可以重新加载当前日期的统计数据。

### 查看历史数据

1. 使用日期选择器选择历史日期
2. 点击 "查询" 按钮
3. 页面会显示该日期的统计数据

### 数据分析

**高异常频率公众号**：
- 异常次数 > 10 次：红色标签
- 异常次数 > 5 次：橙色标签
- 异常次数 ≤ 5 次：蓝色标签

**建议操作**：
1. 对于异常频率高的公众号，检查是否被封禁或限制
2. 查看异常URL，确认是否为文章已删除
3. 根据统计数据调整采集策略

## 数据保留期

- **统计数据**: 保留 30 天
- **详细日志**: 保留 7 天
- **最近日志**: 显示最近 20 条

## 后端要求

确保后端服务已正确配置：

1. **Redis 配置**
   ```yaml
   # config.yaml
   redis:
     url: redis://:password@host:port/db
   ```

2. **API 端点**
   - `/api/env-exception/stats` - 获取统计数据
   - `/api/env-exception/today` - 获取今日统计

3. **权限要求**
   - 需要管理员权限访问
   - 需要登录认证

## 故障排查

### 问题 1: 页面显示 "暂无环境异常记录"

**可能原因**：
- Redis 未配置或连接失败
- 确实没有环境异常发生
- 选择了没有数据的日期

**解决方案**：
1. 检查 Redis 配置和连接状态
2. 运行诊断工具：`python diagnose_redis.py`
3. 尝试查询其他日期

### 问题 2: 无法访问页面

**可能原因**：
- 没有管理员权限
- 未登录

**解决方案**：
1. 使用管理员账号登录
2. 检查用户权限设置

### 问题 3: 数据加载失败

**可能原因**：
- 后端服务异常
- API 接口错误
- 网络连接问题

**解决方案**：
1. 检查后端服务是否正常运行
2. 查看浏览器控制台错误信息
3. 检查网络连接

## API 接口

如果需要通过 API 直接访问数据：

### 获取今日统计

```bash
curl -X GET "http://localhost:8001/api/env-exception/today" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

### 获取指定日期统计

```bash
curl -X GET "http://localhost:8001/api/env-exception/stats?date=2024-01-01" \
  -H "Authorization: Bearer YOUR_TOKEN"
```

## 集成建议

### 1. 定期检查

建议每天查看一次异常统计，及时发现和处理问题。

### 2. 设置告警

可以基于统计数据设置告警规则：
- 当日异常次数超过阈值时发送通知
- 特定公众号频繁异常时提醒

### 3. 数据导出

可以通过 API 导出数据用于进一步分析：

```python
import requests

response = requests.get(
    "http://localhost:8001/api/env-exception/today",
    headers={"Authorization": "Bearer YOUR_TOKEN"}
)

data = response.json()
# 处理或导出数据
```

## 最佳实践

1. **定期监控**: 每天查看异常统计，及时发现异常趋势
2. **优先处理**: 优先处理异常频率高的公众号
3. **数据备份**: 定期导出重要统计数据
4. **优化策略**: 根据统计数据优化采集策略
